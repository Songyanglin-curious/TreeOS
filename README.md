# 升维（Ascend）

Ascend 是一个借助 AI 形成持续增益循环的人机协作系统。

它的目标不是单纯让 AI 产出更多内容，而是让人在每一轮与 AI 的交互中：

- 把问题推进到更可继续的状态
- 保住主线，减少失控
- 让理解能够跨时间累积
- 把一次次提升固化为新的能力基线

## 核心理念

AI 的生成能力近乎无限，但人的精力、注意力和控制力是有限的。
Ascend 关注的核心问题是：

**如何在无限生成中，为有限主体保存可控主线。**

所以 Ascend 不是普通的聊天增强器，也不是单点功能工具，而是一套面向持续推进与持续积累的协作框架。

## 最小回合协议

Ascend 当前最稳定的底层骨架是：

### Expand

扩展问题空间，生成候选、张力、变量、反例与方向。

### Select

从当前候选中做出推进选择，决定先走哪条，不走哪条。

### Crystallize

把本轮结果固化成可继续的状态，让未来还能接上。

这三步共同解决三个核心问题：

- Expand：防止看得不全
- Select：防止分支失控
- Crystallize：防止无法累积

## 当前系统分层

Ascend 目前可以理解为三层结构：

### 1. 元循环层

- Expand
- Select
- Crystallize

### 2. 实例模块层

不同任务场景下的专用模块，例如：

- Expander：思考开图
- Trainer：学习补洞
- ThinkingNode：需求收敛
- Builder：设计与实现
- Coder / Deploy：生成、运行、测试、修复、部署

### 3. 资产层

- Library：模板、反模式、检查清单、决策记录、可复用状态

## ThinkingNode 的位置

ThinkingNode 不是 Ascend 的全部，而是 Ascend 体系中一个相对成熟的强收敛实例。

它擅长把目标压成：

- what
- why
- boundary
- acceptance

也就是把需求转化为可执行、可验证的结构。

## 沉淀的当前理解

Ascend 中的沉淀，不是普通记录或总结。
它的本质是：

**保存可接续状态。**

沉淀的核心不是保存过去，而是让未来还能继续。

## 当前阶段

Ascend 仍处于底层架构逐步冻结的阶段。当前重点不是一次性做成完整系统，而是：

- 明确底层骨架
- 明确模块边界
- 先让最成熟的实例跑稳
- 再由实践推动后续修正

## 一句话定义

**Ascend 是一个借助 AI 形成持续增益循环的人机协作系统。**
