# 1

### 当前冻结判断

这套流程当前先冻结为：

* **A 下发总单给多个 B**
* **B 基于总单下发任务给 C**
* **B 层当前主组织单元先按任务，不按人**
* **任务是跨天持续存在的主对象**
* **每天填报是任务下面的当日记录**
* **应报范围由任务下发时确定的时间范围决定**

---

### 当前定位

当前对象分层先冻结为：

* **总单**
  * A 发起
  * 面向多个 B
* **任务**
  * B 下发给 C
  * 是主业务对象
  * 通常跨多天
  * 单次任务只是时间范围恰好一天的特例
* **任务日报**
  * C 围绕任务在某一天提交的一次记录
  * 提交后即锁定
* **B 当日上送实例**
  * 不是重内容对象
  * 只是把当日 C 的有效记录往 A 上送的一次轻量实例
  * 必须记录本次实际上送的 C 记录集合
  * 缺口由系统反推

---

### 本轮关键变化

本轮真正稳定下来的点有：

* **B 不按人做主轴，先按任务做主轴**
* **任务与每天填报的关系已钉住：1 个任务 → 多天任务日报**
* **B 对 C 没有“通过”按钮**
* **B 只处理异常项：有问题就打回，没打回就默认可上送**
* **B 点上报给 A 时，系统默认带上当天所有“已报且未打回”的记录**
* **A 默认不直接理解 C 的细语义，主要看 B 的结果**
* **应报范围不靠人工判断，而由任务时间范围自动确定**
* **缺口不要求 B 另填一套内容，但系统必须能反推出来**

---

### 下一轮入口

下一轮最自然的入口先冻结为：

也就是后面继续时，可以直接压：

* 是按自然日
* 是否允许中间某天免报
* 是否允许暂停/恢复
* 跨天迟报怎么算到哪一天

当前先冻结到这里。
