# 00 Overview

## 1. Ascend 是什么

Ascend 是一个借助 AI 放大个人生产力、帮助人持续升维的人机协作系统。

在当前阶段，它更具体的落点是：

**一个面向复杂问题持续推进的人机协作架构。**

它要解决的不是“如何让 AI 一次给出更强答案”，而是：

**如何让复杂、易分叉、会反复改写的问题，在多轮协作中仍能持续推进，并保持主线、结构与可控性。**

## 2. Ascend 为什么存在

Ascend 的起点不是单纯做一个工具，而是回应一个根本矛盾：

**AI 的无限生成能力，与人的有限注意力、有限控制力、有限精力之间存在冲突。**

这个冲突会带来几个常见后果：

- 生成越来越多，但主线越来越散
- 问题越聊越复杂，但推进越来越弱
- 看似高产，实际上难以积累
- 一次次对话很强，但人没有形成稳定能力基线

Ascend 要做的，是在这种无限生成中，为有限主体保住可控主线，并把一次次推进逐渐固化成长期增益。

## 3. Ascend 当前阶段的聚焦

Ascend 当前阶段不追求“一次性完整系统”，而是先压实最核心的一层：

**复杂问题推进架构。**

当前主要回答的问题是：

**如何让复杂问题在多轮推进中既能保留复杂性，又能维持可推进性。**

因此，当前阶段聚焦：

- 问题如何持续推进
- 主线如何维持
- 分支如何保留
- 结果如何冻结
- 上位变化后如何做局部回扫

当前阶段不进入：

- UI
- 数据库
- 工程实现
- 自动化编排
- 产品化细节

## 4. Ascend 的总体结构

Ascend 当前采用一个双层耦合结构：

### 4.1 宏观层：问题树

宏观层负责承载问题空间的长期结构。

它的职责不是保存完整思维过程，而是保存：

- 当前主线在哪里
- 出现了哪些问题分支
- 哪些分支在推进
- 哪些分支被挂起
- 哪些节点可能受上位变化影响

所以宏观层本质上是：

**问题空间的结构化地图。**

### 4.2 微观层：推进循环

微观层负责推进当前焦点问题。

当前最小循环为：

**Expand → 人工 Select → Freeze**

它的职责不是维护全局结构，而是围绕某个当前节点完成一轮局部推进，并产出可冻结结果。

所以微观层本质上是：

**对某个具体问题节点的最小推进机制。**

### 4.3 两层关系

两层是耦合的：

- 宏观树提供当前推进位置
- 微观循环围绕该位置推进一轮
- 推进结果写回宏观树
- 写回结果可能带来：
  - 当前节点继续压实
  - 新分支产生
  - 上位理解改写
  - 局部回扫触发

因此，Ascend 不是静态拆树，也不是普通对话流，而是：

**用微观循环驱动宏观问题树持续演化。**

## 5. 当前最小推进循环

当前已冻结的最小循环为：

**Expand → 人工 Select → Freeze**

### Expand

围绕当前节点展开并推进一轮。
它不是固定模板，而是按当前成熟度动态输出。

当前采用 A / B / C 三态：

- A：问题对象未稳定
- B：对象已稳定，但候选池未形成
- C：候选池已形成，可进入 Select

### Select

Select 由人执行，而且必须存在。

它承担：

- 选择当前主线
- 确认保留分支
- 阻止 AI 自动并行扩张
- 维持推进中的主观控制权

### Freeze

Freeze 不是最终结论，也不是完整总结。

它的作用是：

**把当前轮已形成的推进结果冻结成可接续状态。**

这里的“收敛”是弱收敛，只服务于：

- 边界更清
- 当前可继续
- 轮次可接续

## 6. Ascend 的几个关键修正

当前已明确的关键修正包括：

### 1. 认知结构上是树，执行节奏上默认单主线推进

Ascend 承认复杂问题天然树状，但默认每轮只推进一个主分支。
其他分支可以存在，但不默认并行展开。

### 2. 分支机制不是附属品，而是必要组成

复杂问题推进过程中天然会长出：

- 新问题
- 竞争方向
- 暂不推进但重要的旁支

如果没有分支机制，这些内容要么丢失，要么把正文拖重。

### 3. Freeze 只服务接续，不服务完整记录

Ascend 不要求每轮都做厚重沉淀。
Freeze 的目标不是记录全过程，而是让下一轮更容易继续。

### 4. 上位变化时允许局部回扫

当上位定义、核心对象或主线判断发生改变时，Ascend 允许对受影响部分进行局部回扫。
回扫是受控校正，不是全树重做。
