# 04 Micro Loop

## 1. 本章目的

本章用于定义 Ascend 的微观推进循环。

它回答的是：

- 当前最小推进循环是什么
- 为什么这个循环必须存在
- Expand、人工 Select、Freeze 各自负责什么
- A / B / C 三态分别代表什么
- 为什么这里的“收敛”是弱收敛
- 一轮微观循环的最小产出是什么

本章不讨论：

- 宏观树如何承载整体结构
- 分支如何在树上挂载
- 回扫如何在结构上展开
- 运行时模块提示词如何具体写

这些内容分别由后续章节定义。

---

## 2. 微观循环的定义

Ascend 的微观层，不负责维护整个问题空间，而只负责：

**围绕当前节点，完成一轮最小推进。**

当前已冻结的最小循环为：

**Expand → 人工 Select → Freeze**

这三步共同构成 Ascend 当前阶段最小可用的推进单元。

它们的关系不是并列功能，而是一个有顺序的最小闭环：

1. Expand 打开并压实当前问题
2. 人工 Select 决定当前主线
3. Freeze 把本轮结果冻结成可接续状态

因此，微观循环的本质不是“一次性解决问题”，而是：

**让一个仍然不完整、仍然可能改写的问题，也能稳定往前推进一轮。**

---

## 3. 为什么最小循环必须是这三步

Ascend 当前不是为了让 AI 自动一路生成到底，而是为了让人机协作中的推进既能持续，又能受控。

如果缺少这三步中的任意一步，系统都会失衡。

### 3.1 只有 Expand，没有 Select

会发生的问题是：

- AI 会持续生成
- 方向会越来越多
- 当前主线会逐渐失控
- 看起来推进很快，实际上缺乏真正选择

因此，Expand 不能独立构成完整循环。

### 3.2 只有 Expand 和 Select，没有 Freeze

会发生的问题是：

- 当前轮虽然做了选择
- 但结果没有被压成可接续状态
- 下一轮仍然容易重新散开
- 每轮都要重新恢复上下文

因此，选择之后还必须有 Freeze。

### 3.3 只有 Select 和 Freeze，没有 Expand

会发生的问题是：

- 没有足够问题空间被展开
- 没有真实候选可供判断
- Select 会变成伪选择
- Freeze 会冻结空壳

因此，Select 与 Freeze 都依赖 Expand 先把当前问题推进到可选、可冻的程度。

所以，当前最小循环必须是：

**Expand → 人工 Select → Freeze**

---

## 4. Expand 的定义

Expand 是微观循环的第一步。

它的任务不是直接收敛，而是：

**围绕当前节点展开并推进一轮，使当前问题更清楚、更可判断、更接近可选择状态。**

Expand 不是固定模板输出，而是随当前问题成熟度动态变化。

当前已冻结为 **A / B / C 三态动态输出**。

---

## 5. A / B / C 三态

### 5.1 状态 A：问题对象未稳定

状态 A 表示：

- 当前讨论对象还不清楚
- 关键层次还混在一起
- 事实、结构、视图、责任、边界可能尚未拆开
- 真正该回答的问题还没有被压实

此时 Expand 的主要任务不是列方案，而是：

- 压对象
- 拆层次
- 暴露隐藏张力
- 找到真正的当前焦点

状态 A 的目标不是给出答案，而是：

**让当前问题开始真正成形。**

### 5.2 状态 B：对象已稳定，但候选池未形成

状态 B 表示：

- 当前问题对象已经基本明确
- 当前到底在讨论什么，已经能说清
- 关键分歧点开始显现
- 但还没有形成可真实比较的候选池

此时 Expand 的主要任务是：

- 继续压关键分歧点
- 让候选逐渐成熟
- 避免过早进入 Select

状态 B 的目标不是立即拍板，而是：

**让选择点真正长出来。**

### 5.3 状态 C：候选池已形成，可进入 Select

状态 C 表示：

- 已形成两个或以上候选
- 候选处于同一层级
- 候选回答的是同一个核心问题
- 候选差异明确，且可真实比较

此时 Expand 的主要任务不再是继续大规模展开，而是：

- 简洁呈现候选池
- 明确当前选择点
- 为人工 Select 做准备

状态 C 的目标是：

**把问题从“继续展开”推进到“可以进入人工选择”。**

---

## 6. Expand 的本质

Expand 的本质不是“尽量多想”，而是：

**让当前节点从模糊、混杂、未定状态，走到更清楚、更可判断的状态。**

因此，Expand 的产出可能是：

- 当前节点更清楚了
- 当前问题被重新压实了
- 当前关键分歧点暴露出来了
- 当前候选池初步形成了
- 当前节点长出了新的重要分支
- 当前推进结果改写了上位理解

Expand 不保证每轮都生成候选，也不保证每轮都把问题推到 Select。
它的真正任务只是：

**让当前节点向“更可推进”移动一格。**

---

## 7. 人工 Select 的定义

Select 是微观循环的第二步。

它的作用不是附带动作，而是 Ascend 的关键控制点。

当前已明确：

**Select 必须存在，而且由人来执行。**

这意味着，Ascend 不接受“AI 自己展开、自己判断、自己拍板”构成完整循环。

### 7.1 为什么 Select 必须由人执行

因为 Select 承担的是：

- 选择当前主线
- 确认哪些分支暂不推进
- 阻止 AI 自动并行扩张
- 把问题推进的控制权保留在人手里

如果 Select 交给 AI 自动完成，系统会很容易出现：

- 主线悄悄漂移
- 候选尚未成熟就被提前拍板
- 未选中的内容被无声丢失
- 问题空间被模型风格主导，而不是由人的目标主导

因此，Select 不是一个可省略步骤，而是：

**Ascend 用来维持主观控制权的关键门。**

### 7.2 Select 的作用边界

Select 做的是：

- 选择当前主线
- 定义当前不跟的分支
- 确认保留分支如何存在

Select 不做的是：

- 再次大规模 Expand
- 把整轮重新分析一遍
- 生成沉淀文本
- 维护树结构

它的角色是“拍板”，不是“重做”。

---

## 8. Freeze 的定义

Freeze 是微观循环的第三步。

它的作用不是总结全文，也不是输出最终答案，而是：

**把当前轮已形成的推进结果，冻结成可接续状态。**

这里的重点不是“记录得多完整”，而是：

- 当前已经成立了什么
- 当前主线现在押在哪里
- 哪些内容被保留但当前不跟
- 下一轮最自然从哪里继续

因此，Freeze 服务的是：

**接续。**

而不是：

- 漂亮总结
- 完整复盘
- 全过程存档
- 终局定稿

### 8.1 为什么 Freeze 必须存在

如果没有 Freeze，微观循环就很容易发生：

- 当前轮推进结果无法稳定保留
- 下一轮重新散开
- 轮次之间接不起来
- 沉淀越来越依赖全文回看

Freeze 的存在，是为了把一轮局部推进压成一个可继续的状态，而不是让每一轮都像从头开始。

---

## 9. 为什么这里的“收敛”是弱收敛

Ascend 当前明确采用的是：

**弱收敛。**

这意味着，Freeze 只要求：

- 边界比上一轮更清楚
- 当前节点可以继续
- 轮次之间能接上

它不要求：

- 问题被彻底定义完
- 所有边界被穷尽
- 所有分支都被处理
- 形成严格验收式终态

因此，Ascend 的 Freeze 不是强收敛系统中的“完成态”，而是：

**当前轮足够稳定，可以暂时停住，并为下一轮提供继续基础。**

弱收敛的意义在于：

**承认复杂问题在很长一段时间内都处于未完成但可推进的状态。**

---

## 10. 一轮微观循环的最小产出

Ascend 的一轮微观循环，不以“最终答案”作为默认产出。

它更典型的产出，是以下几类之一：

### 10.1 当前节点更清楚了

即：

- 当前到底在讨论什么更明确了
- 关键对象或边界被压实了

### 10.2 当前节点出现了明确候选

即：

- 当前真正的选择点长出来了
- 候选已具备可比较条件

### 10.3 当前节点完成了主线选择准备

即：

- 人工 Select 已有足够依据
- 主线可以被确认

### 10.4 当前节点长出了新的重要分支

即：

- 有新的独立问题显形
- 它不适合本轮顺手解决
- 需要被保留为后续结构内容

### 10.5 当前节点改写了上位理解

即：

- 当前推进结果不只是局部变化
- 它会反过来影响更上位节点
- 后续可能触发局部回扫

因此，微观循环的产出不是固定模板，而是：

**当前节点在复杂性中被真正往前推了一轮。**

---

## 11. 微观循环与宏观树的关系

微观循环虽然只围绕当前节点工作，但它不是孤立发生的。

一轮循环结束后，结果可能对宏观层产生结构影响，例如：

- 当前节点继续压实
- 新分支出现
- 保留项需要写入
- 某个保留分支升级为后续正式节点
- 上位理解被改写，从而触发局部回扫

因此，微观循环的职责虽然是局部推进，但它的结果会反过来驱动宏观树演化。

不过，这种关系只说明两层耦合，并不意味着：

- 微观循环直接承担树管理
- 树直接替代当前轮推进

两层仍然必须分层处理。

---

## 12. 本章结论

Ascend 的微观层不是一次性解题器，而是一个最小推进循环。

当前已冻结的最小循环是：

**Expand → 人工 Select → Freeze**

其中：

- Expand 负责把当前节点推进到更清楚、更可判断的状态
- Select 负责由人保住主线与控制权
- Freeze 负责把本轮结果冻结成可接续状态

A / B / C 三态描述的是 Expand 的成熟度阶段，而弱收敛描述的是 Freeze 的目标强度。

因此，微观循环可以浓缩为一句话：

**在不要求问题彻底完成的前提下，让当前节点在一轮内获得真实推进，并把推进结果稳定交给下一轮。**
