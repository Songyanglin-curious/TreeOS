# TreeOS核心

> **TreeOS 的唯一目标是实现思考收敛**

- AI 负责发散：补全隐含信息，提供候选方案
- 人类负责收敛：确认节点有效性，决定结构走向

约束和组织问题的最小单元 我称之为 [思考原子](./references/思考单元.md) ,它以 **是什么**、**为什么**、**边界**、**验收标准** 四个单元来回答和约束一个问题,这四点 **正交**且**完备**；**思考单元**具有**自相似性**和**递归特性**;一个问题由一堆子问题有层级地组合起来的树状结构或流程图，对每一个拆解出来的子问题，再次套用这个**思考原子**并回答四个单元。

## 工作流总览（四步循环）

1. 选定一个“当前问题”（一个节点）
2. 让 AI 生成该节点的  [ThinkingAtom](./references/ThinkingAtom.ts)（what/why/boundary/acceptance）
3. 你做“收敛审查”（改到可接受为止）
4. 判断要不要拆：
   * 可直接验收 → 叶子节点
   * 不可直接验收 → 让 AI 给 children（子节点），进入下一轮
