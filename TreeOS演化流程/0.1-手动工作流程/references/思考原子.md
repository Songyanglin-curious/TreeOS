## 一、正交性

### 1. 是什么（what）

作用：定义对象本体
解决问题：我们在讨论的“对象”是什么
风险控制：避免语义漂移

它约束“概念清晰度”。

### 2. 为什么（why）

作用：定义存在的理由
解决问题：为什么要做它
风险控制：避免无意义扩展

它约束“价值合理性”。

### 3. 边界（boundary）

作用：定义不做什么
解决问题：范围在哪里停止
风险控制：避免无限膨胀

它约束“复杂度”。

### 4. 验收标准（Acceptance Criteria）

作用：定义完成判定
解决问题：何时算完成
风险控制：避免永远未完成

它约束“终止条件”。

四者之间没有重复维度：

* “是什么”不等于“为什么”
* “为什么”不等于“边界”
* “边界”不等于“验收”
* “验收”不等于“定义”

它们分别覆盖：

> 本体 / 价值 / 范围 / 终止

这是一个完整闭环。

## 二、可收敛

1. 没有“是什么” → 无法讨论
2. 没有“为什么” → 容易偏离价值
3. 没有“边界” → 会无限发散
4. 没有“验收” → 永远无法结束

它会强制每个节点必须回答：

* 是否有价值
* 是否范围清晰
* 是否可验证

## 三、可验证

关键在于“验收标准”。

如果验收标准是：

* 可观察
* 可判断
* 可二值化（满足 / 不满足）

那整个单元就是可验证的。

如果验收是模糊表述，例如：

> “效果更好”

那就不可验证。

因此，这个模型成立的前提是：

> 验收必须具体、可判定。
